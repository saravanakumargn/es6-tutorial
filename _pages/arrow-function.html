There are two benefits to arrow functions.

  1. Shorter syntax => compared to function expressions.
  2. this is picked from the lexical scope (also called static scope).

Arrow functions are always anonymous. The examples are below:

Function without any parameters

{% highlight javascript %} var intro = () => "Welcome"; {% endhighlight %}

The above code is equivalent of the below:

{% highlight javascript %} var intro = function intro() { return "Welcome"; }; {% endhighlight %}

Using Arrow function with map

{% highlight javascript %} const arr = [1, 2, 3]; const squares = arr.map(x => x * x); {% endhighlight %}

The above code is equivalent of the below:

{% highlight javascript %} var arr = [1, 2, 3]; var squares = arr.map(function (x) { return x * x; }); {% endhighlight %}

Function with 1 parameter

{% highlight javascript %} var multiplyBy2 = value1 => value1 * 2; console.log(multiplyBy2(4)); // prints 8 {% endhighlight %}

The above code is equivalent of the below:

{% highlight javascript %} var multiplyBy2 = function multiplyBy2(value1) { return value1 * 2; }; console.log(multiplyBy2(4)); // prints 8 {% endhighlight %}

Function with more than 2 parameters

{% highlight javascript %} var add = (value1, value2) => value1 + value2; console.log(add(10, 20)); // prints 30 {% endhighlight %}

The above code is equivalent of the below:

{% highlight javascript %} var add = function add(value1, value2) { return value1 + value2; }; console.log(add(10, 20)); // prints 30 {% endhighlight %}

this reference

The source of this is an important distinguishing aspect of arrow functions:

  • Traditional functions have a dynamic this; its value is determined by how they are called.
  • Arrow functions have a lexical this; its value is determined by the surrounding scope.

The following variables are all lexical inside arrow functions:

  • arguments
  • super
  • this
  • new.target

An arrow function does not create its own this context, so this has the original meaning from the enclosing context.

Working with this until traditional way

{% highlight javascript %} function Person() { var _this = this; this.count = 0; setInterval(function () { console.log(_this.count++); // |this| properly refers to the person object }, 1000); } {% endhighlight %}

The above code is equivalent of the below:

{% highlight javascript %} function Person(){ this.count = 0; setInterval(() => { console.log(this.count++); // |this| properly refers to the person object }, 1000); } {% endhighlight %}

Returning object literals

The object literal returned by this arrow function is like below:

{% highlight javascript %} const f = x => ({ bar: 123 }) {% endhighlight %}

The above code is equivalent of the below:

{% highlight javascript %} var f = function f(x) { return { bar: 123 }; }; {% endhighlight %}
